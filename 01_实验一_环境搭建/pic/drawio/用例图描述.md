# 学院宣传工作智能协同管理平台_用例图描述

## 参与者

| 参与者 | 说明 |
|---|---|
| 申请人 | 学院部门、社团、班级或活动负责人，负责提交宣传需求并确认成果。 |
| 老师/主任 | 负责审核宣传需求、统筹任务拆分与部门分配、复审成果并查看统计。 |
| 部长 | 负责接收部门任务、安排干事、确认截止时间并进行成果初审。 |
| 干事 | 具体执行宣传任务，接收提醒、提交成果，并按审核意见修改成果。 |
| 系统管理员 | 维护用户、部门、任务类型和提醒规则等基础数据。 |
| 定时任务 | 作为系统触发者，根据截止时间自动产生节点提醒。 |

## 主要用例

| 用例 | 主参与者 | 简要说明 |
|---|---|---|
| 提交宣传需求 | 申请人 | 填写活动名称、时间、地点、宣传类型、联系人、截止时间和附件材料。 |
| 校验申请信息 | 系统 | 检查必填项和格式，是“提交宣传需求”的包含用例。 |
| 结构化申请信息 | 系统 | 将表单或自由文本整理为标准字段，是“提交宣传需求”的包含用例。 |
| 审核需求 | 老师/主任 | 判断需求是否合理、材料是否完整，并记录审核意见。 |
| 驳回/要求补充材料 | 老师/主任 | 当申请不完整或不合理时扩展“审核需求”，申请人修改后重新提交。 |
| 拆分子任务 | 老师/主任/系统 | 按宣传类型生成摄影、视频、推送、新闻稿、海报等任务。 |
| 分配部门 | 老师/主任 | 将任务分配给摄影部、新媒体部、新闻部、美工部等部门。 |
| 安排干事 | 部长 | 选择执行干事并确认截止时间，包含“检查技能与负载”。 |
| 接收节点提醒 | 干事/部长/主任/老师 | 定时任务根据截止时间发送提醒和逾期提醒。 |
| 提交成果 | 干事 | 上传照片、视频、文案、新闻稿、海报或网盘链接，包含“上传附件/链接”。 |
| 共享拍摄素材 | 干事 | 摄影任务完成后上传照片素材，并通知依赖照片的推送、新闻等任务。 |
| 提交推送/新闻草稿预审 | 干事/老师/主任 | 推送或新闻任务可先提交文字草稿、标题和排版框架给老师预审，照片就绪后再补图完善。 |
| 标记任务依赖 | 老师/主任/系统 | 拆分任务时记录推送、新闻等任务对摄影照片的依赖关系。 |
| 初审成果 | 部长 | 对干事成果进行初审；不通过时扩展到“修改成果”。 |
| 复审成果 | 老师/主任 | 对初审通过的成果进行复审；不通过时扩展到“修改成果”。 |
| 确认成果 | 申请人 | 对最终成果进行确认，确认后包含“归档素材”。 |
| 查看数据统计 | 老师/主任 | 查看申请数量、任务数量、完成率、逾期率、返修率和传播效果。 |
| 管理基础数据 | 系统管理员 | 管理用户、部门、任务类型和提醒规则。 |

## 基本流程

1. 申请人提交宣传需求，系统校验并结构化申请信息。
2. 老师/主任审核需求；通过后进入任务拆分，驳回则返回申请人修改。
3. 系统或老师/主任根据宣传类型拆分子任务，主任分配到对应部门。
4. 系统记录摄影任务与推送、新闻任务之间的素材依赖；可并行的推送、新闻任务先完成草稿并进入预审。
5. 摄影任务上传照片素材后，系统通知依赖任务补充照片并完善最终稿。
6. 部长安排干事并确认截止时间，系统向干事发送任务通知和节点提醒。
7. 干事完成任务并提交成果，部长初审，老师/主任复审。
8. 审核不通过时，干事按意见修改成果并重新提交。
9. 复审通过后申请人确认成果，系统归档素材并更新统计数据。

## 后置结果

完成后系统保存宣传需求、任务记录、审核记录、提醒记录、成果资产和统计指标，保证宣传工作过程可追踪、成果可沉淀、数据可分析。
